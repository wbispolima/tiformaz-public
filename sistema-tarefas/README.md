# Sistema de Tarefas FORMAZ

## Visão geral

Esta seção descreve como a Formaz organiza o atendimento de demandas por meio do sistema de tarefas (Redmine).
O foco é padronizar categorias, tipos de tarefa, prioridades, responsáveis e fluxos de atendimento.

---

## Conceitos básicos

Sugestão de termos a padronizar:

- **Projeto**: agrupamento lógico de demandas de um mesmo cliente ou área.
- **Demanda**: necessidade registrada pelo cliente (interno ou externo).
- **Tarefa**: unidade de trabalho executada para atender uma demanda.
- **Tipo de tarefa**: correção, melhoria, incidente, suporte, consulta, implantação etc.
- **Categoria**: agrupamento por área funcional ou módulo.
- **Prioridade**: baixa, média, alta, crítica.
- **Status**: novo, em análise, em desenvolvimento, em homologação, concluído, cancelado etc.

## Regras gerais de uso

Pontos a detalhar:

- Quem pode abrir tarefas (interno/cliente).
- Campos obrigatórios ao abrir uma tarefa (título, descrição mínima, anexos, categoria, tipo, prioridade).
- Como registrar esforço/tempo gasto.
- Como deve ser o encerramento da tarefa (registro de solução, validação, comunicação com o cliente).
